Vagrant Override Default SSH Port

  • Sep 01, 2015
  • Modified on Sep 01, 2015
  • 2 min read
  • Categories: post
  • Tags: #Devops

How to overide vagrant default ssh port.

Vagrant forward port 2222 on host to 22 on guest for ssh. What more important is that it will override whatever you set for port forwarding of port 22 without the mark id:"ssh".

Let me make it clear,the following port forwarding will not take effort, vagrant will say there is a port conflict, and set a random port on host for you instead of 2223.


  # Create a forwarded port mapping which allows access to a specific port
  # within the machine from a port on the host machine. In the example below,
  # accessing "localhost:8080" will access port 80 on the guest machine.
  config.vm.network "forwarded_port", guest: 22, host: 2223, guest_ip: "10.0.2.14"
  config.vm.network "forwarded_port", guest: 80, host: 8083, guest_ip: "10.0.2.14"
  #config.vm.network "forwarded_port", guest: 24, host: 3007, guest_ip: "10.0.2.14"

what vagrant shows may look like this one


==> ubuntu64: Fixed port collision for 22 => 2222. Now on port 2200.

but the following configure will work well.


  # Create a forwarded port mapping which allows access to a specific port
  # within the machine from a port on the host machine. In the example below,
  # accessing "localhost:8080" will access port 80 on the guest machine.
  config.vm.network "forwarded_port", guest: 22, host: 2223, guest_ip: "10.0.2.14", id:"ssh"
  config.vm.network "forwarded_port", guest: 80, host: 8083, guest_ip: "10.0.2.14"
  #config.vm.network "forwarded_port", guest: 24, host: 3007, guest_ip: "10.0.2.14
