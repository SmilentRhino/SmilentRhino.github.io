Do Not Init A Class With An Empth List As A Default Argument

  • Sep 16, 2015
  • Modified on Sep 16, 2015
  • 1 min read
  • Categories: post
  • Tags: #Devops

I encounter a class initilazation problem which many newbie would expirence.


class TestExample:
    def __init__(self, a= []):
        self.a = a


def test_example():
    a_example = TestExample()
    b_example = TestExample()
    a_example.a.append("a") # this will not only change a in a_example but also a in b_example
    print("value of a in b_example:",b_example.a) # b_example.a is not an empty list

if __name__ == "__main__" :
    test_example()

Here is the output:


(env27)[vagrant@vagrant_10 test0]$ python test1.py
('value of a in b_example:', ['a'])
(env27)[vagrant@vagrant_10 test0]$

DO NOT INIT A CLASS WITH A EMPTY LIST AS A DEFAULT ARGUMENT

For details, please refer to (http://effbot.org/zone/default-values.htm)[http://effbot.org/zone/default-values.htm]
